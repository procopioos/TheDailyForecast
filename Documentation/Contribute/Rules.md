# Contribution Guide

## Introduction
Welcome to The Daily Forecast project. Your contribution is invaluable. This guide outlines the standards and protocols required to ensure the maintenance of the codebase's integrity, quality, and architectural coherence.

## Standard Contribution Workflow
To integrate your work into the project, please follow this protocol:

1.  **Branch Isolation:** All contributions must be developed on a dedicated feature branch derived from the main codebase.
2.  **Code Quality:** Contributors must ensure strict adherence to established project conventions, including coding style, naming protocols (e.g., variable casing), and structural patterns utilized throughout the repository.
3.  **Testing Protocol:** All newly introduced features or fixes must be accompanied by robust and passing automated tests (unit/integration tests) that validate the intended behavior and prevent regressions.
4.  **Commit History:** Contributors must utilize clear, atomic commit messages that succinctly describe both *what* the change is and *why* it was made.

## Agentic AI Usage Policy
If generative Artificial Intelligence (AI) tools were utilized during the development process, adherence to the following protocol is **mandatory**:

*   **Contributor Responsibility:** The contributing individual assumes absolute ownership and liability for the entire submission, regardless of AI assistance.
*   **Critical Code Review:** All code suggested or generated by an AI must undergo a meticulous, line-by-line review by the contributor to ensure perfect alignment with project architecture and intent.
*   **Validation:** Comprehensive testing must be executed against the AI-generated segments to verify functional correctness and stability.
*   **Security & Risk Mitigation:** The contributor must explicitly confirm that the introduced code is secure, introduces no vulnerabilities (including injection flaws or resource leaks), and poses zero unintended risks to the project's stability.
